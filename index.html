Reactive programming with Rx

A quick story

So what is reactive programming then?

🤷‍♀️

- paradigma de programación declarativa - data streams - propagación del cambio - FRP: las ventajas de FP en RP -> inmutabilidad, predictibilidad, testing

And why should I care?

😴

- Manejar data streams de forma más eficiente (y generar código más legible) - UI: animaciones, eventos, transformación de datos API - Server side: concurrencia, back pressure, circuit breakers, etc - Manejo de errores

Behold... Rx

An API for asynchronous programming with observable streams
Java, Scala, C#, C++, Clojure, JavaScript, Python, Groovy, JRuby, Swift, Kotlin...
- Observer pattern - Iterator pattern - programación funcional

Building blocks

Observable 👀

- Source of data - Creado con la clase observable - next para emitir valores - error para emitir error - complete para terminar - devuelve la función de unsubscribe - subscribe para leer valores - (cold vs hot)

Observable: from 👀

- array, eventos, ajax, fichero, etc

Subscription 📝

- Asíncrona - se puede desuscribir - onComplete, onError

Data operators 🚦🔪

- Métodos funcionales de colecciones sobre datos asíncronos - Ultra eficiente - uso de pipe

Observable operators 🚦🔪

- ¿Que pasa si nuestros operadores devuelven otros observables? - Ejemplo: detalles de una lista de resultados
