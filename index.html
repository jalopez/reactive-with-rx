So what is Reactive Programming?

- paradigma de programación declarativa - data streams - propagación del cambio - FRP: las ventajas de FP en RP -> inmutabilidad, predictibilidad, testing

Introducing Rx

An API for asynchronous programming with observable streams
- Observer pattern - Iterator pattern - programación funcional - en múltiples lenguajes

Why should I learn it?

- Manejar data streams de forma más eficiente (y generar código más legible) - UI: animaciones, eventos, transformación de datos API - Server side: concurrencia, back pressure, circuit breakers, etc - Manejo de errores

Building blocks

  • Streams ⚡️
  • Observables 👀
  • Operators 🚦
Observable: - Source of data - Creado con la clase observable - next para emitir valores - error para emitir error - complete para terminar - devuelve la función de unsubscribe - subscribe para leer valores - (cold vs hot)
Suscripción: - Asíncrona - se puede desuscribir - onComplete, onError
Observable.from: - array, eventos, ajax, fichero, etc
